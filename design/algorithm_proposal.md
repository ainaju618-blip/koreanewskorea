## 1. 제안 배경

### 1.1 현황 분석 요약

Korea NEWS는 **전남/광주 26개 기관**의 보도자료를 자동 수집하여 통합 제공하는 지역 뉴스 자동화 플랫폼입니다. 현재 다음과 같은 기술적 기반이 구축되어 있습니다:

| 구분 | 현황 | 완성도 |
|------|------|--------|
| 스크래퍼 시스템 | 26개 지역 완성 | 100% |
| 데이터 검증 시스템 | 4단계 검증 로직 | 90% |
| 개인화 인프라 | 부스트/가중치/행동추적 | 70% |
| 콘텐츠 랭킹 | 기본 시간 기반 점수 | 50% |
| SEO/E-E-A-T | 기자 프로필, 메타태그 | 60% |

### 1.2 도출된 과제

현재 시스템 분석 결과, 다음 **5가지 핵심 개선 영역**이 도출되었습니다:

1. **콘텐츠 랭킹 알고리즘의 단순성** — 시간 기반 선형 점수만 적용
2. **유사 콘텐츠 중복 문제** — 제목 완전 일치만 탐지
3. **개인화 데이터 단절** — 디바이스 간 동기화 불가
4. **API 성능 한계** — 캐싱 레이어 부재
5. **E-E-A-T 심화 부족** — 구조화 데이터 미흡

## 2. 제안 개요

### 2.1 프로젝트 목표

> **"데이터 수집을 넘어, 스마트한 콘텐츠 전달 시스템 구축"**

본 제안서는 Korea NEWS 플랫폼의 알고리즘을 고도화하여 다음 목표를 달성하고자 합니다:

| # | 목표 | 핵심 지표(KPI) |
|---|------|---------------|
| 1 | 콘텐츠 노출 최적화 | 속보 노출 시간 **50% 단축** |
| 2 | 중복 콘텐츠 제거 | 유사 기사 중복률 **30% 감소** |
| 3 | 사용자 참여 향상 | 페이지 체류 시간 **20% 증가** |
| 4 | 시스템 성능 개선 | API 응답 시간 **50% 단축** |
| 5 | 검색 노출 강화 | 구글 E-E-A-T 점수 향상 |

## 3. 상세 제안 내용

### 3.1 Phase 1: Quick Wins (즉시 적용)

#### 3.1.1 시간 감쇠 함수 개선

| 항목 | 내용 |
|------|------|
| **현재 방식** | 선형 감쇠 (100 - 시간*2) |
| **제안 방식** | 지수 감쇠 (반감기 12시간) |
| **기대 효과** | 속보성 강화, 골든 타임 활용 극대화 |
| **난이도** | ⭐ (1시간 이내) |

**현재 vs 제안 비교:**

| 발행 후 시간 | 현재 점수 | 제안 점수 | 차이 |
|-------------|----------|----------|------|
| 1시간 | 98점 | 94점 | -4 |
| 6시간 | 88점 | 71점 | -17 |
| 12시간 | 76점 | 50점 | -26 |
| 24시간 | 52점 | 25점 | -27 |
| 48시간 | 10점 | 6점 | -4 |

> **핵심:** 초반 노출 집중 → 후반 롱테일로 자연스러운 순환

---

#### 3.1.2 유사도 기반 중복 탐지

| 항목 | 내용 |
|------|------|
| **현재 방식** | 제목 완전 일치 (exact match) |
| **제안 방식** | Jaccard 유사도 70% 이상 시 중복 판정 |
| **기대 효과** | 유사 기사 중복률 30% 감소 |
| **난이도** | ⭐⭐ (반나절) |

**탐지 가능 케이스:**

| 기사 A | 기사 B | 현재 | 제안 |
|--------|--------|------|------|
| "나주시, 2025년 예산안 확정" | "나주시 2025년 예산안 확정!" | ❌ 미탐지 | ✅ 탐지 |
| "광주시 공공버스 노선 개편" | "광주시, 공공버스 노선 개편 발표" | ❌ 미탐지 | ✅ 탐지 |

---

#### 3.1.3 인덱스 최적화

| 항목 | 내용 |
|------|------|
| **제안 인덱스** | `idx_posts_status_published`, `idx_posts_region_date` |
| **기대 효과** | 쿼리 성능 50% 향상 |
| **난이도** | ⭐ (10분) |

**추가 SQL:**

```sql
-- 인기 기사 조회 최적화
CREATE INDEX idx_posts_status_published ON posts(status, published_at DESC)
    WHERE status = 'published';

-- 지역별 최신 기사 조회
CREATE INDEX idx_posts_region_date ON posts(region, published_at DESC);
```

### 3.2 Phase 2: Core Enhancement (핵심 고도화)

#### 3.2.1 Hot Score 알고리즘 도입

| 항목 | 내용 |
|------|------|
| **목적** | 실시간 인기도 반영 |
| **알고리즘** | Reddit/Hacker News 스타일 Hot Score |
| **기대 효과** | 사용자 참여 기반 콘텐츠 노출 |
| **난이도** | ⭐⭐⭐ (1~2일) |

**계산 공식:**

```
Hot Score = (조회수) / (발행 후 시간 + 2)^1.8
```

**적용 시나리오:**

| 기사 | 조회수 | 발행 시간 | 시간 점수 | Hot Score | 최종 순위 |
|------|--------|----------|----------|-----------|----------|
| A | 500 | 2시간 전 | 80 | 41.3 | **1위** |
| B | 50 | 1시간 전 | 95 | 16.7 | 3위 |
| C | 200 | 3시간 전 | 70 | 20.0 | 2위|

| **기대 효과** | 개인화 정확도 향상 |

## 4. 기대 효과

### 4.1 정량적 효과

| 지표 | 현재 | 목표 | 개선율 |
|------|------|------|--------|
| 속보 노출 시간 | 발행 후 ~30분 | 발행 후 ~15분 | **50% 단축** |
| 유사 기사 중복률 | ~15% | ~10% | **30% 감소** |
| 페이지 체류 시간 | 1분 30초 | 1분 48초 | **20% 증가** |
| API 응답 시간 | 200ms | 100ms | **50% 단축** |

### 4.2 정성적 효과

- ✅ **사용자 경험 향상** — 개인화된 콘텐츠 추천으로 만족도 증가
- ✅ **운영 효율화** — 자동화된 중복 탐지로 관리 부담 감소
- ✅ **검색 노출 강화** — E-E-A-T 개선으로 구글 검색 순위 향상
- ✅ **확장성 확보** — 캐싱 및 배치 처리로 트래픽 증가 대응

## 6. 결론

Korea NEWS 플랫폼은 **데이터 수집 인프라**가 100% 완성된 상태입니다.

1. **Phase 1**로 즉시 개선 효과를 체감하고

3. **Phase 3**로 장기적 경쟁력을 확보

할 수 있습니다.
