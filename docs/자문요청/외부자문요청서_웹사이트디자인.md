### 2.2 현재 서비스 기능

#### A. 핵심 기능 - 뉴스 서비스
```
┌─────────────────────────────────────────────────────────────┐
│  1. 자동 뉴스 수집 시스템                                    │
│     - 지자체 공식 홈페이지에서 보도자료 자동 수집            │
│     - 광주광역시, 전라남도, 나주시, 목포시, 순천시, 여수시   │
│     - 1일 평균 50-100건 신규 기사 수집                       │
│                                                              │
│  2. 카테고리 기반 뉴스 분류                                  │
│     - 정치, 경제, 사회, 문화, 교육, 복지, 환경, 관광 등      │
│     - AI 기반 자동 카테고리 분류 시스템                      │
│                                                              │
│  3. 이미지 자동 매칭 시스템                                  │
│     - 10,000+ 키워드 데이터베이스                            │
│     - 기사 제목/본문 분석 → 적합한 이미지 자동 할당          │
│     - Unsplash, Cloudinary CDN 활용                          │
└─────────────────────────────────────────────────────────────┘
```

#### B. 부가 기능 - 점술/운세 서비스
```
┌─────────────────────────────────────────────────────────────┐

│     - 64괘, 384효 완전 데이터베이스                          │

│                                                              │
│  2. 오늘의 운세                                              │
│     - 일간/주간/월간/연간 운세                               │
│     - 사용자 생년월일 기반 개인화                            │
└─────────────────────────────────────────────────────────────┘
```

- 사용자 접속 시 IP 기반으로 해당 지역 뉴스 우선 표시
- 지역 전환 기능 제공

- 각 시/도 하위에 시/군/구 단위 뉴스 제공
- 예: 전라남도 → 나주시, 목포시, 순천시, 여수시, 광양시 등 22개 시/군

## 4. 자문 요청 사항

### 4.1 정보 구조(IA) 설계

**질문 1**: 17개 시/도 + 250개 이상 시/군/구의 뉴스를 효과적으로 구조화하는 방법은?
```
현재 고민:
- 깊이 3단계 (전국 → 시/도 → 시/군/구)가 적절한가?
- 권역별 그룹핑 (수도권, 충청권 등)이 필요한가?
- GNB(Global Navigation Bar)에서 어떻게 표현할 것인가?
```

**질문 2**: 지역 뉴스와 전국 뉴스의 혼합 비율은?
```
현재 고민:
- 메인 페이지에서 전국 뉴스 vs 사용자 지역 뉴스 비율
- 사용자가 원하는 정보를 빠르게 찾도록 하는 UI
- 다른 지역 뉴스도 자연스럽게 노출하는 방법
```

### 4.2 UI/UX 디자인

**질문 3**: 지역 선택 UI의 최적 형태는?
```
현재 고려 중인 옵션:
A. 드롭다운 메뉴 방식
B. 지도 기반 인터랙티브 선택
C. 탭/버튼 그룹 방식
D. 검색 기반 자동완성 방식

각 방식의 장단점과 추천 방식을 자문 부탁드립니다.
```

**질문 4**: 뉴스 카드 디자인의 최적화
```
현재 고민:
- 썸네일 이미지 비율 (16:9 vs 4:3 vs 1:1)
- 카드에 표시할 정보 (제목, 요약, 날짜, 출처, 카테고리)의 우선순위
- 리스트 뷰 vs 그리드 뷰 vs 매거진 레이아웃
- 무한 스크롤 vs 페이지네이션
```

**질문 5**: 모바일 경험 최적화
```
현재 고민:
- 하단 네비게이션 바 구성 요소
- 스와이프 제스처 활용 방안
- 푸시 알림과 연계한 UI
- 오프라인 읽기 기능 UI
```

### 4.3 시각적 아이덴티티(VI)

**질문 6**: 브랜드 컬러 전략
```
현재 상황:
- 기존: 파란색 (#3B82F6 계열)
- 지역별 차별화된 컬러가 필요한가?
- 다크 모드 지원 시 컬러 전략

자문 요청:
- 뉴스 포털에 적합한 메인 컬러
- 보조 컬러 및 액센트 컬러 체계
- 지역별 컬러 코드 필요 여부
```

**질문 7**: 타이포그래피 전략
```
현재 상황:
- 본문: Pretendard 400/500
- 제목: Pretendard 600/700
- 영문: Inter

자문 요청:
- 뉴스 가독성을 높이는 폰트 조합
- 제목/본문/캡션 등 계층 구조
- 모바일에서의 최적 폰트 크기
```

### 4.4 차별화 전략

**질문 8**: 기존 뉴스 포털(네이버, 다음)과의 차별점
```
현재 고민:
- 지역 뉴스 특화 포털로서의 아이덴티티
- 공공 데이터 기반 신뢰성 강조 방법
- 커뮤니티 기능 추가 여부
- 사용자 참여 유도 방안 (댓글, 제보, 공유)
```

**질문 9**: 점술/운세 서비스의 위치
```
현재 고민:
- 뉴스 포털에 점술 서비스가 어울리는가?
- 별도 서브 도메인으로 분리해야 하는가?
- 어울린다면 어떻게 자연스럽게 통합할 것인가?
- 수익화 모델과의 연계
```

### 4.5 접근성 및 성능

**질문 10**: 접근성(A11y) 준수 방안
```
자문 요청:
- WCAG 2.1 AA 등급 충족을 위한 체크리스트
- 고령자/시각장애인을 위한 UI 고려사항
- 키보드 네비게이션 최적화
- 스크린 리더 호환성
```

**질문 11**: 성능 최적화 디자인
```
현재 지표:
- LCP (Largest Contentful Paint): 2.5s 목표
- FID (First Input Delay): 100ms 이하 목표
- CLS (Cumulative Layout Shift): 0.1 이하 목표

자문 요청:
- 이미지 로딩 전략 (lazy loading, placeholder)
- 스켈레톤 UI 적용 범위
- 애니메이션 사용 가이드라인
```
