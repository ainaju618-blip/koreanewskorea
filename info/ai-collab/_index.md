# AI 협업 가이드

> **용도:** Claude와 Gemini가 함께 작업할 때 참조
> **대상:** Claude, Gemini (Antigravity)

---

## 핵심 규칙

```
┌─────────────────────────────────────────────────────────────┐
│  "인포 참조해" = info/_index.md부터 읽기                     │
│                                                              │
│  1. info/_index.md 읽기 (진입점)                            │
│  2. 상황에 맞는 _catalog.md 찾기                            │
│  3. 키워드 매칭으로 해당 파일 읽기                          │
│  4. 해결책 적용                                              │
└─────────────────────────────────────────────────────────────┘
```

---

## 역할 분담

| AI | 역할 | 주요 작업 |
|----|------|----------|
| **Claude** | 기획/관리 | 작업 지시서, Git push, 문서 관리 |
| **Gemini** | 실행/검증 | 코드 작업, 브라우저 테스트, 스크래퍼 |

---

## 소통 채널

```
.ai-collab/
├── TASK.md        # 작업 지시 (Claude → Gemini)
├── DONE.md        # 완료 보고 (Gemini → Claude)
└── QUESTION.md    # 질문/논의 (양방향)
```

### TASK.md 형식
```markdown
## [URGENT] 작업 제목

### 목표
- 무엇을 해야 하는지

### 파일
- 수정할 파일 목록

### 완료 조건
- [ ] 체크리스트
```

### DONE.md 형식
```markdown
## [완료] 작업 제목

### 수행 내용
- 무엇을 했는지

### 변경 파일
- 변경된 파일 목록

### 테스트 결과
- 스크린샷 또는 결과
```

---

## 에러 참조 방법

### 주인님이 "인포 참조해" 하면:

**1단계: 진입점**
```
info/_index.md 읽기
```

**2단계: 카탈로그**
```
에러면 → info/errors/_catalog.md
가이드면 → info/guides/_catalog.md
```

**3단계: 키워드 검색**
```
_catalog.md에서 현재 문제 키워드 찾기
→ 해당 파일 경로 확인
```

**4단계: 해결**
```
해당 파일 읽고 해결책 적용
```

---

## 예시 시나리오

### 이미지 에러 발생 시
```
1. info/_index.md → "에러니까 errors/_catalog.md"
2. errors/_catalog.md → "이미지" 키워드 → image-missing.md
3. errors/scraper/image-missing.md 읽기
4. 해결책 적용
```

### 새 스크래퍼 만들 때
```
1. info/_index.md → "가이드니까 guides/_catalog.md"
2. guides/_catalog.md → "새 스크래퍼" → new-scraper.md
3. guides/scraper/new-scraper.md 읽기
4. 절차대로 진행
```

---

## 주의사항

1. **전체 파일 읽지 않기** - 필요한 파일만 읽어서 토큰 절약
2. **카탈로그 먼저** - 바로 파일로 가지 말고 카탈로그에서 검색
3. **관련 파일 확인** - 각 파일 하단의 "관련" 섹션 참고
