## Part 1: 현재 상태 비판적 분석

### 1.1 헤더 구조 문제점

**현재 헤더 구조 (총 ~200px):**
```
┌────────────────────────────────────────────────────┐
│ Layer 1: Top Utility Bar (32px)                    │  ← 필요한가?
├────────────────────────────────────────────────────┤
│ Layer 2: Logo Zone (70px)                          │  ← 너무 높음
├────────────────────────────────────────────────────┤
│ Layer 3: Main Navigation (55px)                    │  ← 적정
├────────────────────────────────────────────────────┤
│ Layer 4: News Ticker (45px)                        │  ← 필요한가?
└────────────────────────────────────────────────────┘
```

**문제점:**
| 항목 | 현재 | 문제 | 영향 |
|------|------|------|------|
| 전체 높이 | ~200px | **뷰포트의 15-20% 차지** | 콘텐츠 노출 감소 |
| Top Utility Bar | 소셜링크+날짜+로그인 | 지역사이트에 불필요한 요소 | 시각적 노이즈 |
| Logo Zone | 로고+브랜딩+광고 | 좌/우 요소 불균형 | 시각적 산만함 |
| News Ticker | 속보 티커 | 광주 전용 뉴스 아님 | 본사와 중복 |

**핵심 비판:**
> 지역 사이트는 본사의 축소판이 아니다.
> 본사 헤더를 그대로 가져오면 지역 정체성이 사라진다.

### 1.2 GNB 메뉴 문제점

**현재 GNB:**
```
Home | News TV | [DB에서 로드하는 카테고리들...]
```

**문제점:**
1. **정체성 혼란**: DB에서 본사 카테고리를 그대로 가져옴
2. **News TV?**: 광주 지역사이트에 이게 왜 필요?
3. **메가메뉴 과잉**: 지역사이트에 22개 지역 드롭다운은 오버킬
4. **하드코딩 혼재**: Home, News TV는 하드코딩, 나머지는 DB

**핵심 비판:**
> 광주 시민이 이 사이트에서 원하는 것:
> "광주 뉴스를 빠르게 보고 싶다"
>
> 현재 GNB가 이 니즈를 충족하는가? **No.**

### 1.3 메인 페이지 레이아웃 문제점

**현재 레이아웃:**
```
┌─────────────────────────────────────────────────────┐
│                 HERO SECTION                        │
│   [Slider 8칸] [Ad + Article 4칸]                  │
│              (500px 높이)                           │
├─────────────────────────────────────────────────────┤
│                                                     │
│   [NewsGrid "광주" 72%]          [Sidebar 28%]     │
│                                                     │
│   [NewsGrid "광주 최신" 72%]                       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**문제점:**
| 항목 | 현재 | 문제 |
|------|------|------|
| Hero Slider | 6개 지역 순환 | 광주만 보여주면 되는데 왜 전남 전체? |
| NewsGrid 중복 | "광주" + "광주 최신" 2개 | 같은 categorySlug인데 왜 2번? |
| Sidebar | 일반적인 "많이 본/최신" | 광주 특화 콘텐츠 없음 |
| 정보 계층 | 모호함 | 무엇이 가장 중요한지 불명확 |

**핵심 비판:**
> "광주" 카테고리 기사를 두 번 보여주는 이유가 무엇인가?
> 첫 번째는 6개, 두 번째는 4개...
> 사용자가 구분할 수 있는가? **구분 불가능.**

---

### 1.4 콘텐츠 구조 문제점

**현재 콘텐츠 소스:**
- `posts` 테이블에서 `category.eq.광주` 또는 `region.eq.gwangju` 필터링

**문제점:**
1. **카테고리 부재**: 정치/경제/사회/문화 구분 없이 모두 "광주"
2. **시간 기반만**: 최신순 정렬만 있고, 주제별 정리 없음
3. **하이라이트 부재**: 에디터 픽, 주요 뉴스 구분 없음

**핵심 비판:**
> 광주 시민이 "광주 정치 뉴스"를 보고 싶으면?
> 현재 구조에서는 불가능. 모든 것이 "광주"라는 하나의 버킷에 담겨 있음.

## Part 3: 새로운 메뉴 구조

### 3.1 GNB 메뉴 (Type A - 광역시용)

**제안안:**
```
┌─────────────────────────────────────────────────────┐
│  홈  │  정치  │  경제  │  사회  │  문화  │  오피니언  │
└─────────────────────────────────────────────────────┘
```

**메뉴 정의:**

| 메뉴 | 슬러그 | 설명 | 콘텐츠 소스 |
|------|--------|------|-------------|
| 홈 | `/` | 메인 페이지 | - |
| 정치 | `/category/politics` | 시정, 의회, 선거 | 태그 기반 |
| 경제 | `/category/economy` | 산업, 기업, 고용 | 태그 기반 |
| 사회 | `/category/society` | 교육, 복지, 환경 | 태그 기반 |
| 문화 | `/category/culture` | 문화, 관광, 스포츠 | 태그 기반 |
| 오피니언 | `/category/opinion` | 칼럼, 사설, 독자투고 | 수동 분류 |

**Why 6개?**
- 7개 이상: 인지 과부하
- 5개 이하: 너무 단순, 분류 부족
- 6개: 골디락스 존 (연구 기반)

### 3.2 제거할 요소

| 요소 | 제거 이유 |
|------|-----------|
| News TV | 광주 전용 콘텐츠 없음, 본사 링크로 대체 가능 |
| 지역 메가메뉴 | 광주 사이트에서 22개 지역 필요 없음 |
| CosmicPulse | 이미 제거됨 ✅ |
| Top Utility Bar 소셜 | 클릭률 0.1% 미만 예상 |

### 3.3 유지할 요소

| 요소 | 유지 이유 | 위치 변경 |
|------|-----------|-----------|
| 기자 로그인 | 기자 접근 필요 | 푸터 또는 햄버거 메뉴 |
| 구독 | CTA 유지 | 사이드바 또는 푸터 |
| 검색 | 필수 기능 | GNB 우측 유지 |

## Part 6: 기술적 구현 방향

### 6.1 DB 스키마 변경

**현재:**
- `posts.category` = "광주" (지역명)
- `posts.region` = "gwangju" (영문 코드)

**제안:**
- 신규 컬럼: `posts.topic` (정치/경제/사회/문화/기타)
- 신규 테이블: `post_topics` (다대다 관계)

```sql
-- 기존 posts 테이블에 추가
ALTER TABLE posts ADD COLUMN topic VARCHAR(20) DEFAULT 'general';

-- 또는 다대다 관계
CREATE TABLE post_topics (
  post_id UUID REFERENCES posts(id),
  topic VARCHAR(20) NOT NULL,
  PRIMARY KEY (post_id, topic)
);
```
