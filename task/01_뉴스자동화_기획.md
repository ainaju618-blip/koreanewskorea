# 뉴스 자동화 시스템 (News Automation) 기획

## 1. 개요
- **목적:** 1인 미디어의 인력 부족 한계를 기술(AI, Bot)로 극복하여, 운영자는 '단순 노동'에서 해방되고 '데스크(판단)' 역할에 집중함.
- **핵심 가치:** 업무 효율화, 콘텐츠의 양적/질적 확보, 저작권 이슈 회피.

## 2. 주요 기능 상세

### A. 이메일 보도자료 자동 처리 (Real-time)
> 언론사 메일로 들어오는 보도자료를 실시간으로 기사화.
- **입력:** 이메일 수신 (제목, 본문, 첨부파일[HWP, PDF])
- **처리 프로세스:**
    1. **감지:** 봇이 메일함을 24시간 감시.
    2. **추출:** 첨부파일을 다운로드하고 텍스트 내용 추출.
    3. **AI 가공:** 추출된 텍스트를 AI에게 전달하여 '기사 형태'로 요약 및 재작성. (제목 뽑기, 리드문 작성)
    4. **저장:** 관리자 페이지에 '발행 대기(Draft)' 상태로 저장.
- **출력:** 관리자 대시보드에서 '승인' 버튼 클릭 시 즉시 발행.

### B. 타 매체 기사 재구성 (Back-filling)
> 사이트 초기 콘텐츠 확보 및 공공기관 소식 보강.
- **입력:** 타겟 지역 신문사(광주 드림, 무등일보 등)의 기사 URL.
- **처리 프로세스:**
    1. **수집:** 2024년 9월 19일 이후의 주요 뉴스(광주/전남/나주/교육/AI) 수집.
    2. **재구성 (Rewriting):** 수집된 본문을 AI를 통해 **"전혀 다른 문체로 다시 쓰기"** 수행. (저작권 침해 방지)
        - *지침:* "건조하고 차분한 어조로", "팩트 위주로", "언론사 말투로"
    3. **저장:** 재작성된 기사를 DB에 적재.

### C. 해외 뉴스 AI 번역/재가공 (Global AI News)
> *확정 전략: 단순 번역을 넘어선 '재창조'로 저작권 이슈 해소*
- **소스:** AI News, Unite.AI, TechCrunch, MIT Tech Review (RSS 피드 활용)
- **저작권 전략:** 영문 기사를 한국어로 번역하고 요약하는 과정에서 **"새로운 창작물(2차 저작물)"** 성격을 띠게 함. 원본 링크는 '출처'로 명시.
- **처리 프로세스:**
    1. **RSS 감지:** 해외 사이트 RSS 피드 주기적 확인.
    2. **AI 번역+요약:** 
        - 단순 직역 지양.
        - "한국 독자가 이해하기 쉽게", "우리나라 실정에 맞게" 의역 및 해설 추가.
    3. **자동 발행:** 별도 승인 없이 '해외 토픽' 카테고리에 자동 등록 (옵션).

## 3. 운영 워크플로우 (User Scenario)
1. **[Bot]** 밤새 이메일과 타사 뉴스를 수집/가공하여 DB에 쌓아둠.
2. **[발행인]** 아침에 관리자 페이지 접속 -> "어제 들어온 보도자료 20건" 확인.
3. **[발행인]** 제목을 쓱 훑어보고 중요한 것 5개만 선택 -> **[발행 승인]** 클릭.
4. **[System]** 사이트 메인 페이지에 즉시 노출.
