# 효사 명구 데이터 작업지시서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트 | AI 주역 해석 서비스 |
| 작업 목표 | 384효 명구 데이터 구축 |
| 목적 | LLM 할루시네이션 방지 (AI가 "생성"하지 않고 "선택"만 하도록) |
| 검증자 | Claude AI |

---

## 작업 분배

| 개발자 | 담당 괘 | 괘 번호 | 효 개수 |
|--------|---------|---------|---------|
| 1번 | 건괘 ~ 동인괘 | 1-13 | 78효 |
| 2번 | 대유괘 ~ 대축괘 | 14-26 | 78효 |
| 3번 | 이괘 ~ 건괘(蹇) | 27-39 | 78효 |
| 4번 | 해괘 ~ 간괘 | 40-52 | 78효 |

## 저장 위치

```
d:\cbt\yu-1\backend\app\data\yao_phrases\
├── _template.json          # 템플릿 (참고용)
├── gua_01_qian.json        # 건괘

├── ...
└── gua_64_weiji.json       # 미제괘
```

**파일명 규칙**: `gua_XX_name.json`
- XX: 두 자리 숫자 (01, 02, ... 64)
- name: 괘 영문명 (소문자)

---

## 공통 작업 규칙

### 1. 톤 규칙 (필수)

```
✅ 허용 표현:
- "~하라", "~하느니라", "~이로다", "~이니라"
- "때가 이르지 않았느니라"
- "하늘이 이르기를..."

❌ 금지 표현:
- "~해 보세요", "~하면 좋겠어요"
- "~일 수도 있어요", "~것 같아요"
- 이모지, 이모티콘
- 현대 상담사 어조
```

### 2. 명구 개수 규칙

| 항목 | 개수 |
|------|------|
| universal_phrases (공통 명구) | 3-5개 |
| category_phrases (카테고리별) | 2-4개씩 |
| closing_phrases (마무리) | 2-3개 |

### 3. 방향 일관성 규칙

| 효사 방향 | 명구 톤 |
|-----------|---------|
| 상승 | 적극, 진행, 도약 |
| 정체 | 대기, 인내, 준비 |
| 하강 | 주의, 후퇴, 경계 |

## 검증 요청 방법

1. 작업 완료 후 `yao_phrases/` 폴더에 저장
2. Claude에게 검증 요청: "gua_XX_name.json 검증해줘"
3. 검증 통과 시 완료 / 수정 필요 시 피드백 반영
